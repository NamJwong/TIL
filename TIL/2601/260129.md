  - 정리 후보의 성격별로 구분해서 알려주겠다고 함. -> Insight, Must-Keep, Refactor(보완 필요)

### AI 에이전트란?

- 목표를 달성하기 위해 자율적으로 행동하는 AI 시스템
- 단순히 TIL을 주고 후보를 만들어주는 입출력 프로그램은 AI 에이전트가 아님.

### 사이드 프로젝트 방향성

1. 첫 시작은 단순히 TIL -> 정리 후보 입출력 프로그램 만들기
2. AI 에이전트를 만들어보고 싶은 니즈를 반영, TIL 레포지토리를 관리해주는 에이전트로 만들기. 더 나아간다면 성장 관리를 해줄 수도 있을 듯.
   - TIL 올릴 때 내용 정리 돕기
   - 매주 TIL -> 지식 정리 PR 올려주기
   - 추가되는 내용에 따라 폴더 구조 추천
   - 주기적으로 업데이트 필요한 내용 제안
   - 새로운 프로젝트/학습 제안

3. 에이전트 프레임워크 살펴보기
4. 일반화 할 수 있는 포인트를 찾아보고 있으면 일반화 해서 주변에 공유해보기.
   - 없는데 억지로 하지 않기. 이젠 새로운 프로젝트 쉽게 만들 수 있으니까 더 좋은 문제 정의를 찾아서 새로운 프로젝트 하면 됨.

### 클로드 추천 로드맵

1단계: 기본 TIL 분석기 (시작)

- TIL 텍스트 입력받기
- AI로 핵심 주제/키워드 추출
- 지식 항목 후보 제안

2단계: 자동화 추가

- 파일에서 TIL 자동 읽기
- 정리된 내용 마크다운으로 저장
- 여러 TIL 배치 처리

3단계: 지능형 분류

- 카테고리 자동 분류 (개발/디자인/비즈니스 등)
- 태그 자동 생성
- 중요도 평가

4단계: 지식 연결 (에이전트화)

- 과거 지식과 연관성 찾기
- 유사 주제 자동 링크
- 지식 그래프 생성

5단계: 능동적 관리

- 주기적으로 지식 창고 리뷰
- 업데이트 필요한 내용 제안
- 학습 패턴 분석 및 추천
