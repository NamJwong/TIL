### acc 프리코스에서 배운 점/성장한 점

1. 코드 작성할 때 끝도 없이 생겨나는 고민에 대한 기준점이 생겨, 중요한 고민에 집중할 수 있게 됨.
   - 고민 예시
     - 방법론에 대한 고민 (ex 폴더 아키텍처, 로딩 처리 방법)
     - 각종 추상화 기준 (ex 컴포넌트의 책임)
   - 기준점: 이 코드의 본질이 무엇인가? 그걸 잘 보여주는 코드인가?
2. 추상화 관련 안티 패턴 교정
   - 결합도가 부족한 코드 (aka. ABAB 패턴)
     - ABAB 패턴에서 A끼리 결합을 시키지 않고, A-1을 따로 추상화 해버림.
   - 예측가능성이 떨어지는 코드 (aka. 징검다리의 다리 사이가 멀게 느껴지는 코드)
     - 컴포넌트 이름과 prop이 서로 멀게 느껴짐.
3. 코드에 대한 생각 중 ‘진짜 내 것’이 아닌 것이 많았다는 걸 자각하고, ‘진짜 내것’을 만들기 위해 노력하게 되었음.
   - 다양한 사람들과의 대화가 도움이 됨.
4. 추상화, 본질에 집중하는 건 결국 인생에 있어서 필요한 일. 인생 가치관까지 고민해보는 계기가 되었음.

### acc 인터뷰

- 준비: 긴장돼서 스스로에게 이것저것 자문하면서 긴장되는 이유, 긴장하지 않아도 되는 이유를 생각함
  - 좋은 방법이었다고 피드백 받음
- 배운 것
  - 잘하고 있는 것
    - 어떻게 해야 하는지 좋은 방향성을 알고 있다
    - 어떤 배움에 대해 나의 케이스로 연결시키는 걸 잘한다
  - 부족한 것
    - 알고 있는 방향성, 배움을 체화하는 것
  - 자기 확신을 가지고 말하기 → 경험 기반으로 얘기하는 게 도움이 될 것
  - 불확실한 것에 도전할 때 0 또는 1이 아니라 0.n으로 접근하기
  - 학습한 것을 압축(추상화)하는 시간이 필요하다
- 아쉬웠던 것
  - 피드백 요청을 했으면 좋았겠다.
- 액션 아이템
  - 주간 회고에 ‘배운 점’ 추가해서 압축하기
