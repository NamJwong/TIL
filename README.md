# TIL

## 작성 목적 및 규칙

### 목적: 개발자로서의 꾸준한 성장

- 어떤 것을 기록하는가?
  - 새롭게 배운 점
  - 확실하게 나의 지식으로 만들고 싶은 정리하고 싶은, 애매하게 알고 있는 지식
  - 개발자로서의 고민 (계속 업데이트 해나간다.)
  - 사이드 프로젝트 개발 일지
- 작성하는 행위 자체에 매몰되지 않는다.
  - 성장에 의의가 있는 것을 기록한다.
  - 내용의 질에 집착하지 않는다.
    - 다만, 다른 사람이 볼 수 있다는 여지를 두며 **다른 사람도 이해할 수준**으로 작성하자. 학습 내용에 대한 신뢰 수준을 지키고, 추후 TIL 스터디나 피드백 요청 등 공개 학습의 효과를 얻을 수 있는 여지를 두기 위함이다.
- 작성하는 지식의 깊이를 따지지 않는다. 나의 성장에 의의가 있는지에 집중한다.

## 작성 절차

1. 당일의 날짜를 따서 브랜치를 만든다. (ex 251214)
2. 위 규칙에 따라 TIL을 작성한다.
3. 하루 동안의 TIL 작성이 끝나면 작성 내용을 검토해 PR을 올리고 머지한다.

\*PR 오픈 이유: 일별 학습 기록 추적 용이하게 하기 위해
