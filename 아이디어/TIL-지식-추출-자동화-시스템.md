# TIL 지식 추출 자동화 시스템

(클로드가 뽑아준 계획으로 일부 수정 예정)

## 목적

매일 작성하는 TIL(Today I Learned)에서 지식과 인사이트를 주간 단위로 자동 추출하여, 체계적인 지식 저장소를 구축한다.

## 핵심 아이디어

- AI가 매주 TIL을 분석하여 인사이트 추출
- 사용자 정의 가이드라인 문서를 참고하여 개인화된 추출
- 피드백 루프를 통해 가이드라인이 점진적으로 개선됨

---

## 레포지토리 구조

```
your-repo/
├── til/                          # 매일 기록
│   ├── 2026-01-20.md
│   ├── 2026-01-21.md
│   └── 2026-01-27.md
│
├── .knowledge-repo/              # 시스템 설정 (숨김 폴더)
│   ├── guidelines.md             # ⭐ AI 참고 기준 문서
│   └── feedback-history.jsonl    # 피드백 히스토리 (선택)
│
├── programming/                  # 지식 카테고리
│   ├── react-patterns.md
│   └── testing-strategies.md
│
├── design/                       # 지식 카테고리
│   └── ui-principles.md
│
├── productivity/                 # 지식 카테고리
│   └── workflow-tips.md
│
└── insights/                     # 기타 인사이트
    └── 2026-01-week4.md
```

## 가이드라인 문서 예시

`.knowledge-repo/guidelines.md`:

```markdown
# 지식 저장소 관리 가이드라인

## 인사이트 추출 기준

### 포함해야 할 것

- 새로 배운 개념이나 기술
- 문제 해결 과정에서 얻은 교훈
- 반복되는 패턴이나 원칙 발견
- 실무에 적용 가능한 구체적인 방법론
- "아하!" 모먼트, 깨달음

### 제외해야 할 것

- 단순 작업 로그 (예: "오늘 회의 3개 했다")
- 너무 세부적인 구현 디테일
- 맥락 없는 코드 스니펫
- 일회성 트러블슈팅 (반복 가능한 교훈이 없는 경우)

## 카테고리 분류 기준

### programming

- 코딩, 알고리즘, 자료구조
- 프레임워크, 라이브러리 사용법
- 아키텍처, 디자인 패턴
- 성능 최적화

### design

- UI/UX 원칙
- 디자인 시스템
- 사용자 경험 인사이트

### productivity

- 워크플로우 개선
- 도구 사용법
- 시간 관리, 협업 방법

### insights

- 위 카테고리에 속하지 않는 것
- 여러 카테고리에 걸친 통합적 인사이트

## 작성 스타일 선호

- 간결하고 실용적으로
- 미래의 나에게 설명하듯 작성
- 가능하면 구체적인 예시 포함
- 코드는 핵심만 간단히
```

## 워크플로우

### 1. 주간 리뷰 실행

```bash
npm run weekly-review
# 또는
python weekly_review.py
```

### 2. AI 처리 과정

1. `til/` 폴더에서 최근 7일간의 마크다운 파일 읽기
2. `.knowledge-repo/guidelines.md` 가이드라인 로드
3. 각 TIL에서 인사이트 추출
4. 카테고리별로 분류
5. 마크다운 파일 생성 (초안)

### 3. 인터랙티브 피드백

```
추출된 인사이트 (5개):

[1] 📁 programming/react-patterns.md
    "React Server Components에서 데이터 페칭 패턴"
    👍 승인  👎 거부  ✏️ 수정  💬 피드백

[2] 📁 design/ui-principles.md
    "사용자 피드백 루프를 짧게 만드는 UI 설계"
    👍 승인  👎 거부  ✏️ 수정  💬 피드백

선택: 1

[1]을 수정하시겠습니까?
💬 피드백: "이건 너무 구체적인 코드 예시야. 더 high-level 패턴만 남겨줘"

피드백 반영 완료! 다음 가이드라인을 추가할까요?

guidelines.md에 추가할 내용:
---
### 2026-01-31
- "라이브러리 API보다는 설계 원칙과 패턴 위주로"
- "코드 예시는 핵심 3줄 이내"
---

적용하시겠습니까? (y/n): y
✅ 가이드라인이 업데이트되었습니다.
```

### 4. 자동 커밋 (선택)

```bash
git add .
git commit -m "📝 Weekly insights: 2026 W5"
git push
```

## 구현 방안

### 추천: Node.js + Claude API

**필요한 것:**

- Node.js 프로젝트
- Claude API 키 또는 OpenAI API 키
- 간단한 CLI 인터페이스

**장점:**

- 빠르게 프로토타입 가능
- package.json 스크립트로 실행
- 나중에 GitHub Actions로 확장 용이

**기술 스택:**

- Node.js / TypeScript
- Anthropic SDK (Claude API)
- inquirer (인터랙티브 CLI)
- gray-matter (마크다운 파싱)

### 대안 1: Python + LangChain

- 복잡한 프롬프트 관리에 유리
- 피드백 히스토리 구조화 쉬움

### 대안 2: GitHub Actions (완전 자동화)

- 매주 일요일 자동 실행
- PR로 제안 → 검토 후 머지
- 설정 복잡도 높음

## 예상 비용

### Claude API (Sonnet 3.5)

- 주간 TIL ~10,000 tokens 입력
- 인사이트 추출 ~5,000 tokens 출력
- 예상 비용: **주당 $0.10 이하**

### OpenAI API (GPT-4o)

- 비슷한 토큰 수
- 예상 비용: **주당 $0.05 이하**

**→ 월 $1 미만으로 운영 가능**

## 예상 시간 절약

### 현재 (수동)

- 주간 TIL 7개 읽기: 20분
- 인사이트 추출 및 정리: 30분
- 파일 생성 및 커밋: 10분
- **총 60분/주**

### 자동화 후

- 스크립트 실행: 1분
- AI 처리 대기: 2분
- 검토 및 피드백: 10분
- **총 13분/주**

**→ 주당 47분 절약 (78% 효율 개선)**
